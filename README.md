# 精投助手 (JobFit AI)

> AI 驱动的简历-JD 匹配度分析工具 — **不是帮你写简历，而是教你改简历。**

🎯 **在线体验:** [https://vitafix-production.up.railway.app](https://vitafix-production.up.railway.app)

---

## 产品定位

求职者投简历前最焦虑的问题是：**"我这份简历能不能过？哪里需要改？怎么改？"**

市面上的工具要么是模板填空（超级简历/知页简历），要么是 ChatGPT 手动粘贴。精投助手不一样：

| 对比项 | 竞品做法 | 精投助手 |
|--------|---------|---------|
| 修改方式 | AI 自动生成 / 模板填空 | **AI 分析 + 用户自主修改** |
| 分析深度 | 一个总分 | **6 维雷达图 + 逐项比对 + 推理过程** |
| 迭代能力 | 一次性 | **改一段 → 重评分 → 看变化** |
| 用户角色 | 被动接收者 | **主动学习者** |

**核心定位：不替代用户，而是教用户改简历。**

---

## 核心功能

| 功能 | 说明 |
|------|------|
| 📝 JD 输入 | 粘贴文本 / 截图 OCR 识别 |
| 📄 简历输入 | 粘贴文本 / PDF·DOCX·TXT 上传 |
| 🧠 AI 解析 | DeepSeek 自动分板块（教育/经历/项目/技能） |
| 📊 6 维匹配分析 | 匹配度雷达图 + 维度详情卡片 |
| ✅ 需求逐项匹配 | JD 要求 vs 简历事实，逐项比对 |
| 🔍 板块深度分析 | 通用/垂直能力区分 + AI 推理链展示 |
| ✏️ 板块重评估 | 修改简历段落 → AI 重新评分 → 分数变化 |
| 📋 历史管理 | 分析记录保存/重命名/删除 |

### 6 维评分体系

1. **技能匹配** — 硬技能匹配度
2. **经验匹配** — 工作内容相似度
3. **教育匹配** — 学历/专业符合度
4. **项目匹配** — 业务领域相关性
5. **关键词覆盖** — JD 关键词覆盖率
6. **成果量化度** — 数据指标支撑度

## 用户旅程

```
Home → JD输入(粘贴/截图OCR) → 简历输入(粘贴/上传文件)
  → 补充经历(可选) → AI分析 → 查看结果(雷达图+清单+板块)
  → 编辑板块→重评分 → 历史管理
```

## 关键设计决策

### 1. 无登录设计
求职者本来就焦虑，再让他们注册账号会大幅降低转化率。通过设备 ID (`X-User-Id` 请求头) 隔离用户数据，零摩擦启动。后续可平滑升级到手机号登录。

### 2. 异步分析
AI 分析耗时 10-30 秒，采用 **202 Accepted + 轮询** 模式，页面不卡死。后续可升级到 SSE 进一步降低延迟。

### 3. 思维链展示
AI 不只给分数，而是展示完整的推理过程：JD 要求什么 → 简历有什么 → 逐项比对 → 评分理由 → 改进建议。让用户**看到 AI 的思考过程**，增加可信度，也帮助用户理解自己该改什么。

### 4. 通用 vs 垂直能力区分
每个比对项标记为"通用能力"（可迁移技能）或"垂直领域能力"（行业特定知识），帮助用户看清自己的竞争力结构。通用能力匹配度高说明基本功扎实，垂直领域有差距可通过学习弥补。

### 5. 板块重评估
用户修改简历某个板块后，AI 仅对该板块重新评估（轻量级 prompt），快速反馈分数变化。形成"改一段 → 看效果 → 再改一段"的优化闭环。

## 项目结构

```
精投助手demo/
├── web/                     # 前端 (Vue 3 + Vite + Vant)
│   └── src/
│       ├── views/           # 6 个页面
│       ├── components/      # 5 个组件 (分数环/雷达图/维度卡片等)
│       ├── api/             # API 封装
│       └── router/          # Hash 路由
├── server/                  # 后端 (Express + Mongoose)
│   ├── routes/              # 路由
│   ├── controllers/         # 控制器
│   ├── services/            # 业务逻辑 (AI/解析/OCR/文件)
│   ├── models/              # 数据模型
│   └── utils/               # 提示词模板
├── railway.json             # Railway 部署配置
└── package.json             # 根工作流脚本
```

## 本地开发

```bash
# 1. 安装依赖
cd web && npm install
cd ../server && npm install

# 2. 配置环境变量 (server/.env)
# MONGODB_URI=你的MongoDB连接串
# DEEPSEEK_API_KEY=你的DeepSeek API Key

# 3. 启动后端 (端口 3000)
cd server && npm run dev

# 4. 启动前端 (端口 5173，自动代理 /api 到后端)
cd web && npm run dev
```
